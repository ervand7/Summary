## Goal

Become a practical AI / LLM / Agentic AI Engineer who can build real systems, not just “play
with ChatGPT.”

Modern agentic systems usually involve LLMs that can plan, call tools, keep state, collaborate
across specialized agents, and complete multi-step workflows. OpenAI describes agents in a
similar way in its Agents SDK docs.

---

# Month 1 Roadmap

## Stage 1 — Core LLM Foundations

Learn:

1. What an LLM is
2. Tokens and context window
3. System/user/assistant messages
4. Temperature, top_p, max tokens
5. Prompt engineering
6. Structured output
7. JSON mode / schemas
8. Streaming responses
9. Model limitations
10. Hallucinations
11. Cost and latency basics

Practice project:

> Build a simple FastAPI service: `/ask`, which sends user input to an LLM and returns a
structured JSON answer.

---

## Stage 2 — OpenAI / Claude API Basics

Learn:

1. How to call LLM APIs from Python
2. How to manage API keys
3. Sync vs async calls
4. Error handling
5. Rate limits
6. Retries
7. Timeouts
8. Logging prompts and responses
9. Basic observability
10. Prompt versioning

Practice project:

> Build a backend service that receives a text, classifies it, summarizes it, and returns
structured output.

## Stage 3 — Embeddings

Learn:

1. What embeddings are
2. Why text can be converted into vectors
3. Similarity search
4. Cosine similarity
5. Chunking documents

7. Embedding models
8. Storing embeddings
9. Searching relevant chunks
10. Re-ranking basics

Practice project:

## Stage 5 — RAG

RAG means Retrieval-Augmented Generation: first find relevant knowledge, then pass it to the LLM.

Learn:

1. Basic RAG pipeline
2. Document loading
3. Text splitting
4. Embedding generation
5. Retrieval
6. Context injection
7. Answer generation
8. Source citations
9. “I don’t know” behavior
10. RAG evaluation
11. Common RAG bugs
12. Bad chunking
13. Irrelevant retrieval
14. Too much context
15. Missing citations

Practice project:

> Build “Chat with PDF / docs” with citations.

This is one of the most important portfolio projects.

---

## Stage 6 — Tool Calling / Function Calling

Tool calling is where an LLM decides to call your backend functions. OpenAI’s docs treat
tool/function calling as a core pattern for building useful AI apps.

Learn:

1. What tool calling is
2. Function schema
3. Tool arguments
4. Validation
5. Returning tool results to the model
6. Multiple tool calls
7. Tool call errors
8. Safe tool execution
9. Preventing dangerous actions
10. Human confirmation before important actions

Practice project:

> Build an assistant that can call tools: `get_weather`, `search_docs`, `create_ticket`,
`send_email_draft`.

## Stage 9 — LangGraph

LangGraph is very important for agentic AI because it gives more explicit control over agent
workflows, state, branching, retries, and multi-step execution. It is described as an agent
orchestration framework for reliable AI agents.

Learn:

1. Graph-based agent workflows
2. Nodes
3. Edges
4. State
5. Conditional routing
6. Retries
7. Human-in-the-loop
8. Multi-agent flows
9. Supervisor agent
10. Worker agents
11. Persistence
12. Checkpointing

Practice project:

> Build a LangGraph workflow: planner → researcher → critic → final answer generator.

This should be your main portfolio project.

---

## Stage 10 — OpenAI Agents SDK

## Stage 11 — MCP: Model Context Protocol

MCP is a protocol for connecting AI systems to tools and data sources. It was introduced by
Anthropic to reduce custom one-off integrations between models and external systems.

Learn:

1. What MCP is
2. MCP server
3. MCP client
4. Tools over MCP
5. Resources
6. Prompts
7. Local tools
8. Filesystem access
9. GitHub-like integrations
10. Security risks
11. Permission boundaries

Practice project:

> Create a simple MCP server that exposes tools like `search_notes`, `create_task`, `read_file`.
