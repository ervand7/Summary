GC uses the Mark and Sweep algorithm.

========================
FULL GC FLOW (ORDER)
========================

1) Mark Setup – STW (Stop-The-World)
- Very short pause
- All goroutines are stopped
- GC captures root objects (stacks, globals)

------------------------

2) Marking – Concurrent
- Main phase
- Your program CONTINUES running
- GC works in background

Tri-color algorithm:
- Objects = graph
- White → not visited (potential garbage)
- Gray → found, but references not processed yet
- Black → fully processed (alive)

Flow:
- Start from roots
- Move objects: white → gray → black
- Everything reachable becomes black
- Unreachable stays white

------------------------

3) Mark Termination – STW
- Second short pause
- Stop program again
- GC finishes marking
- Ensures no new objects are missed

=> After this step:
- Black = alive
- White = garbage

------------------------

4) Sweep – Concurrent
- No STW
- GC deletes all white objects
- Runs in background (often lazily during allocations)

========================

Summary:
- STW #1 → start Mark
- Concurrent → Marking
- STW #2 → finish Mark
- Concurrent → Sweep

========================

Additionally:
- You can trigger GC manually: runtime.GC()
- GC uses ~25% CPU by default
