* Multiple producers can write to the same Kafka topic simultaneously
* Producers do not block each other; there is no global ordering of messages
* Kafka guarantees ordering only within a single partition
* If producers send messages with the same key (e.g., `user_1`)
  → Kafka uses hash(key) to route them to the same partition
* As a result:
  * all events for `user_1` go to the same partition
  * their order is strictly preserved
* This guarantee can be broken if:
  * the number of partitions changes
  * a custom partitioner is used
  * messages are sent without a key

Interview phrasing:
> Kafka guarantees message ordering per key within a single partition,
even with multiple producers.
