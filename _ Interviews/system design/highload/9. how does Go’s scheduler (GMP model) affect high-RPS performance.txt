Go runs many goroutines on a small number of OS threads.

Why it scales well:
* cheap goroutines
* fast scheduling
* work stealing between CPUs

Problems under high RPS:
* too many runnable goroutines
* blocked goroutines use memory
* more context switching
* lock contention

This increases:
* scheduler overhead
* GC pressure
* latency

> The Go scheduler is efficient,
> but too many goroutines still degrade performance.
