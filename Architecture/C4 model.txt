C4-модель — это способ описывать архитектуру системы от общего к частному,
постепенно увеличивая детализацию.

1. Context — система и её окружение:
   пользователь → наше приложение → внешние сервисы
   (например: User → Banking App → Payment Provider).

2. Container — крупные части системы:
   Web-app, Backend API, Auth-service, Database, Message Broker.

3. Component — устройство одного контейнера изнутри:
   например, внутри Backend API:
   HTTP handlers → business services → repositories → database.

4. Code — детали реализации:
   конкретные структуры, интерфейсы, функции
   (struct OrderService, func CreateOrder()).

Идея C4: сначала показать «что это за система»,
а потом аккуратно раскрывать детали, не смешивая уровни.
