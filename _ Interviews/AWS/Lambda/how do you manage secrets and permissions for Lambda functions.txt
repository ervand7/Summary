I manage Lambda secrets and permissions using IAM roles + AWS Secrets Manager.

Permissions

* Assign an IAM role to the Lambda
* Grant least-privilege policies (S3, DynamoDB, SQS, etc.)
* Lambda gets temporary credentials automatically

Secrets

* Store secrets in AWS Secrets Manager or SSM Parameter Store
* Lambda reads them at runtime
* Encrypt with KMS
* Avoid hardcoding secrets or putting them in plain env vars
