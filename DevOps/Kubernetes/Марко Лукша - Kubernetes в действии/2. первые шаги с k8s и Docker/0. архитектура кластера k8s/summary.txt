 ● Cluster
В Kubernetes, кластер представляет собой распределенную систему, которая управляет и
оркестрирует контейнерами и приложениями в среде с использованием контейнеризации. Он
состоит из набора компьютеров (узлов) и компонентов управления, которые работают вместе
для обеспечения развертывания, масштабирования и управления контейнеризированными приложениями

 ● Node
Узел относится к физическому или виртуальному серверу, на котором запускаются контейнеры.
Узел - это часть кластера Kubernetes, и он предоставляет вычислительные ресурсы для
выполнения приложений в виде контейнеров.
Узлы делятся на 2 типа:
 ● Master
 На нем размещена плоскость управления (control plane), которая контролирует и управляет
 всей сем кластером. Она состоит из:
     ● API Server.
     API Server предоставляет API для взаимодействия с кластером. Команды и запросы от
     администраторов и пользователей отправляются к API Server для управления
     приложениями и ресурсами.

     ● Etcd
     Это распределенное хранилище данных, которое используется для хранения конфигурации
     кластера и состояния.

     ● Controller Manager
     Контроллер-менеджеры мониторят состояние кластера и управляют ресурсами и контроллерами
     (например, ReplicaSet и Deployment).

     ● Scheduler
     Планировщик (Scheduler) решает, на каких узлах кластера следует развернуть новые
     экземпляры приложений (поды), учитывая требования к ресурсам и другие факторы.

 ● рабочие узлы
 Тут выполняются развертываемые приложения. Каждый узел имеет следующие компоненты:
     ● Kubelet
     Это агент, который работает на каждом узле и управляет контейнерами на этом узле.
     Kubelet обеспечивает выполнение контейнеров, исходя из инструкций, заданных в
     объектах Pod, которые описывают, какие контейнеры должны работать на узле.

     ● Docker/Containerd
     Контейнерный движок, такой как Docker или Containerd, отвечает за запуск и управление
     контейнерами на узле. Эти контейнерные движки обеспечивают изоляцию и управление контейнерами.

     ● Kube Proxy
     Это компонент, который управляет сетевой маршрутизацией для Pod на узле. Он обеспечивает
     сетевую доступность между разными Pod и снаружи кластера.

     ● Container Runtime
     Это программное обеспечение, которое фактически выполняет контейнеры. В зависимости
     от конфигурации кластера, это может быть Docker, Containerd, CRI-O или другой
     контейнерный runtime.

     ● Node Status
     Узел также отслеживает свой статус и ресурсы, которые он может предоставить кластеру,
     такие как доступное CPU, память и дисковое пространство.

     ● Pods
     Поды - это минимальные единицы развертывания в Kubernetes, и они работают на узлах.
     Поды могут содержать один или несколько контейнеров и представляют собой логические
     хосты для контейнеров.
