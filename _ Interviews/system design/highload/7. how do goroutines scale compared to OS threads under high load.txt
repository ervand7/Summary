Goroutines are much cheaper than OS threads.

Why:
* tiny dynamic stack
* cheap context switching
* managed by Go runtime

Go runs many goroutines on a small number of OS threads.

But goroutines are not free:
* blocked goroutines still use memory
* too many goroutines increase GC pressure
* unbounded goroutines can degrade the system

> Goroutines scale well, but too many blocked goroutines are dangerous.
