There is no fixed number.
It depends on what one request does.

## Main steps

1️⃣ Understand request cost
* CPU work
* DB/Redis calls
* external APIs
* memory allocations
* latency

---

2️⃣ Estimate concurrency
Use:
Concurrency ≈ RPS × latency

Example:
1000 RPS × 100ms = ~100 concurrent requests

---

3️⃣ Find bottlenecks
Usually:
* CPU
* DB pool
* Redis pool
* network
* memory / GC

---

4️⃣ Load test
Increase RPS gradually and watch:
* p95/p99 latency
* error rate
* CPU
* memory
* GC

Max RPS is where latency or errors suddenly grow fast.

---

## Short interview answer

> I estimate max RPS by understanding request cost, calculating concurrency from
latency, checking system limits, and validating with load testing.
