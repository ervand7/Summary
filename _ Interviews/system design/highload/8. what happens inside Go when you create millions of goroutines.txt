Goroutines are lightweight, but not free.

Each goroutine:
* uses memory (stack + metadata)
* is managed by the Go scheduler
* increases GC work

Under huge load:
* goroutines accumulate
* memory grows
* GC runs more often
* latency spikes

Main danger:
* unbounded goroutine creation

> Millions of goroutines can work,
> but too many blocked goroutines cause memory and GC pressure.
