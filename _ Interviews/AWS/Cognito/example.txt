1. Клиент (браузер/мобилка) отправляет email+password в AWS Cognito.
2. Cognito проверяет данные и возвращает JWT-токен.
3. Клиент вызывает ваш API и кладёт токен в заголовок: `Authorization: Bearer <JWT>`.
4. API Gateway + Cognito Authorizer проверяют токен (подпись/срок) и либо 403, либо
пропускают дальше.
5. Lambda получает запрос уже как “проверенный” и читает `claims` (например `sub`, `email`) →
делает бизнес-логику.
