В транзакции Ethereum содержатся несколько важных полей, которые определяют, как
она будет обработана. Вот основные поля:
1. **Nonce:**
   - Это число, определяющее количество предыдущих транзакций, отправленных с данного
   адреса отправителя. Это поле предотвращает повторную отправку одной и той же транзакции
   (replay attacks).

2. **Gas Price:**
   - Это сумма, которую отправитель готов заплатить за единицу газа (Gas). Газ измеряется
   в Gwei (1 Gwei = 10⁹ Wei). Это поле определяет, сколько ETH будет потрачено на
   выполнение транзакции.

3. **Gas Limit:**
   - Это максимальное количество газа, которое может быть использовано для выполнения
   транзакции. Если для выполнения транзакции требуется больше газа, чем указано в этом
   поле, транзакция завершится с ошибкой (out of gas).

4. **To:**
   - Адрес получателя транзакции. Это поле указывает, на какой адрес отправляется ETH или
   вызов смарт-контракта.

5. **Value:**
   - Сумма ETH, которая отправляется в транзакции. Это поле может быть пустым (0), если
   транзакция не предполагает передачи ETH, а, например, взаимодействие со смарт-контрактом.

6. **Data (или Input):**
   - Это поле содержит данные для вызова смарт-контракта. Например, оно может включать
   функцию, которую нужно вызвать, и параметры, которые нужно передать этой функции. Если
   транзакция — это просто перевод ETH, это поле может быть пустым.

7. **v, r, s (Подпись):**
   - Эти три поля вместе составляют цифровую подпись, которая используется для
   аутентификации отправителя и подтверждения, что транзакция действительно инициирована
   им. Подпись генерируется на основе закрытого ключа отправителя.
   v, r, s - это координаты на кривой алгоритма ECDSA.

8. **Chain ID (опционально):**
   - Идентификатор цепи (Chain ID) используется для предотвращения повторной отправки
   транзакций в разных блокчейнах. Это поле было введено с обновлением EIP-155, и оно
   помогает отличить, к какому блокчейну принадлежит данная транзакция.

### Дополнительные поля в EIP-1559 транзакциях:
С обновлением EIP-1559, появились дополнительные поля:
9. **Max Priority Fee Per Gas (или Tip):**
   - Максимальная сумма, которую отправитель готов заплатить как чаевые валидатору за
   обработку его транзакции. Это ускоряет обработку транзакции.

10. **Max Fee Per Gas:**
    - Максимальная сумма газа, которую отправитель готов заплатить за единицу газа,
    включая базовую плату (base fee) и чаевые (priority fee).

11. **Access List (опционально):**
    - Список адресов и хранилищ, к которым будет осуществлен доступ в ходе выполнения
    транзакции. Это помогает уменьшить потребление газа при доступе к часто используемым
    данным.

Эти поля формируют полную структуру транзакции в сети Ethereum и определяют, как она будет
обрабатываться сетью.
