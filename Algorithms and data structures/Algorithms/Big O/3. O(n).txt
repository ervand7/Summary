Операции с линейной сложностью O(n) выполняются во времени, линейно пропорционально
размеру входных данных (n). Вот несколько примеров операций с O(n) сложностью:

1. **Линейный поиск (Linear Search)**: При линейном поиске элемента в массиве каждый
элемент последовательно сравнивается с искомым значением до тех пор, пока не будет
найден совпадающий элемент или не достигнут конец массива. Линейный поиск имеет
сложность O(n) в худшем случае.

2. **Обход (перебор) массива**: Перебор всех элементов в массиве, чтобы выполнить
какие-либо операции с каждым элементом, также имеет сложность O(n).

3. **Суммирование элементов массива**: Вычисление суммы всех элементов в массиве
требует просмотра каждого элемента, что дает O(n) сложность.

4. **Подсчет элементов**: Подсчет количества элементов в массиве, удовлетворяющих
определенным условиям, также может иметь O(n) сложность.

5. **Копирование данных из одного массива в другой**: Копирование элементов из одного
массива в другой выполняется за O(n) времени, так как каждый элемент должен быть скопирован.

6. **Переворот массива**: Операция, которая инвертирует порядок элементов в массиве,
имеет сложность O(n), так как каждый элемент должен быть перемещен на новую позицию.

7. **Поиск максимального (минимального) элемента**: Поиск максимального или
минимального элемента в массиве требует просмотра каждого элемента, что
дает O(n) сложность.

8. **Фильтрация (отбор) элементов массива**: Создание нового массива, содержащего
только элементы, удовлетворяющие определенным условиям, также может иметь O(n) сложность.

Это только некоторые примеры операций с линейной сложностью O(n). Такие операции
выполняются эффективно для небольших и средних наборов данных, но могут быть
неэффективными для крупных наборов данных, где более эффективные алгоритмы с
меньшей сложностью могут быть предпочтительны.
