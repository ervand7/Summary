В Dockerfile инструкции `ENTRYPOINT` и `CMD` предназначены для определения команд,
которые будут выполняться при запуске контейнера, но они имеют некоторые
ключевые различия:

1. **ENTRYPOINT**:
   - Инструкция `ENTRYPOINT` определяет исполняемую команду, которая будет
   выполнена при запуске контейнера.
   - Если инструкция `ENTRYPOINT` определена, команды, переданные в командной
   строке при запуске контейнера, будут рассматриваться как аргументы и
   передаваться команде, определенной в `ENTRYPOINT`. Это означает, что команда
   из `ENTRYPOINT` будет НЕИЗМЕННОЙ, и аргументы будут дополнять ее.
   - Пример:
     ```Dockerfile
     ENTRYPOINT ["./myapp"]
     ```

2. **CMD**:
   - Инструкция `CMD` также определяет команду, которая будет выполнена при запуске
   контейнера.
   - Однако если команда не указана при запуске контейнера, будет выполнена команда,
   определенная в `CMD`.
   - Если команда указана при запуске контейнера, она ПЕРЕОПРЕДЕЛИТ команду из `CMD`.
   - Пример:
     ```Dockerfile
     CMD ["./myapp"]
     ```

Различия между этими инструкциями сводятся к тому, что `ENTRYPOINT` обычно
используется для определения "основной" команды, которая НЕ ИЗМЕНЯЕТСЯ, и
предназначенной для выполнения одной определенной задачи. А `CMD` обычно
используется для определения команды, которая МОЖЕТ БЫТЬ ПЕРЕОПРЕДЕЛЕНА при
запуске контейнера и может выполнять различные действия в зависимости от контекста.

Примеры использования:
- Если у вас есть контейнер с веб-сервером, `ENTRYPOINT` может определить
команду для запуска веб-сервера, а `CMD` может определить параметры
конфигурации (например, порт и директорию с веб-страницами), которые
МОЖНО ИЗМЕНИТЬ при запуске контейнера.

- Если у вас есть контейнер с базой данных, `ENTRYPOINT` может определить команду
для запуска СУБД, а `CMD` может определить параметры, такие как имя базы данных
и порт, которые также МОГУТ БЫТЬ ПЕРЕОПРЕДЕЛЕНЫ при запуске.
Это позволяет гибко управлять контейнерами и их конфигурацией в зависимости от
потребностей.
