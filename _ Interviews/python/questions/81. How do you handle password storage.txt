Passwords must never be stored in plain text.

* Use strong password hash functions:
  - bcrypt

* Add a unique salt per password
* Use sufficient cost / work factor
* Compare hashes in constant time

Never do:
* Plain hashing (SHA256, MD5)
* Reversible encryption
* Storing passwords in logs or DB dumps

Key idea:
> Store only slow, salted password hashes
