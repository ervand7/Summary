1. **IAM (Identity and Access Management)** — сервис AWS для управления доступом к
ресурсам AWS. Позволяет контролировать, кто (пользователи и роли) имеет доступ к
чему (ресурсам) и на каких условиях (политики доступа).

2. **IAM User (пользователь IAM)** — уникальная сущность в IAM, представляющая
отдельного человека или приложение. Каждому пользователю можно назначить
индивидуальные учетные данные (пароль и/или ключи доступа) и разрешения.

3. **IAM Group (группа IAM)** — коллекция IAM пользователей. Политики доступа могут
назначаться группам, что упрощает управление разрешениями для множества пользователей.

4. **IAM Role (роль IAM)** — набор разрешений, которые можно временно назначить пользователю,
сервису или приложению. Используется для предоставления доступа к ресурсам без необходимости
использования паролей или ключей доступа.

5. **IAM Policy (политика IAM)** — документ, который определяет разрешения. Политики могут
быть назначены пользователям, группам или ролям и определяют, какие действия разрешены или
запрещены для определенных ресурсов.

6. **Managed Policies (управляемые политики)** — политики, которые создаются и управляются AWS.
Эти политики можно назначать пользователям, группам и ролям.

7. **Inline Policies (встроенные политики)** — политики, которые непосредственно привязаны к
одному пользователю, группе или роли. Они не могут быть использованы повторно.

8. **Access Keys (ключи доступа)** — пары ключей (идентификатор ключа и секретный ключ),
используемые для программного доступа к ресурсам AWS.

9. **MFA (Multi-Factor Authentication, многофакторная аутентификация)** — дополнительный
уровень безопасности для учетных записей AWS. Требует использования второго метода
аутентификации (например, временного кода с устройства MFA) помимо пароля.

10. **IAM Console (консоль IAM)** — веб-интерфейс для управления IAM ресурсами, такими как
пользователи, группы, роли и политики.

11. **AWS Organizations** — сервис для управления множеством AWS аккаунтов. Позволяет
централизованно управлять политиками и безопасностью для всех аккаунтов в организации.

12. **STS (Security Token Service)** — сервис для временного предоставления ограниченных
привилегий для доступа к ресурсам AWS. Генерирует временные учетные данные.

13. **Identity Federation (федерация идентичности)** — возможность использовать внешние
системы управления доступом (например, Active Directory) для аутентификации и авторизации в AWS.

14. **Resource-Based Policies (политики на основе ресурсов)** — политики, которые можно
прикрепить непосредственно к ресурсам, таким как S3-бакеты или SNS-топики. Эти политики
определяют, кто и какие действия может выполнять с ресурсом.

15. **IAM Permissions Boundary (граница разрешений IAM)** — функция, позволяющая ограничить
максимальные привилегии, которые могут быть назначены пользователю или роли.

16. **Service Control Policies (SCP)** — политики, используемые AWS Organizations для
управления разрешениями на уровне организации или учетной записи. Эти политики определяют,
какие действия могут выполняться в аккаунтах организации.

17. **Role Assumption (принятие роли)** — процесс временного принятия разрешений роли IAM.
Пользователи или сервисы могут принимать роли для выполнения задач с нужными привилегиями.

18. **IAM Credential Report (отчет о учетных данных IAM)** — отчет, содержащий информацию о
состоянии учетных данных всех IAM пользователей в аккаунте.

19. **Access Analyzer** — инструмент, который помогает выявлять ресурсы, доступ к которым
могут иметь внешние или иные AWS аккаунты, и проверять политики на наличие чрезмерных привилегий.

20. **Least Privilege Principle (принцип наименьших привилегий)** — практика предоставления
минимальных необходимых разрешений для выполнения задач. Помогает минимизировать риски безопасности.
