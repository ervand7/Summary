1. **Amazon EBS (Elastic Block Store)** — блочное хранилище для инстансов
EC2. Обеспечивает постоянное хранилище данных, которое можно подключать к инстансам.

2. **EBS Volume (том EBS)** — единица блочного хранилища, которая может быть
подключена к одному инстансу EC2. Каждый том существует в одной зоне доступности.

3. **EBS Snapshot (снимок EBS)** — резервная копия тома EBS, сохраняемая в
Amazon S3. Снимки можно использовать для восстановления томов или создания новых томов.

4. **Provisioned IOPS (выделенные IOPS)** — тип EBS томов, оптимизированный для
высокой производительности ввода-вывода. Подходит для приложений с высокими
требованиями к производительности.

5. **General Purpose SSD (gp2/gp3)** — тип EBS томов, обеспечивающий сбалансированную
производительность и стоимость. Подходит для большинства рабочих нагрузок.

6. **Throughput Optimized HDD (st1)** — тип EBS томов, оптимизированный для
последовательного ввода-вывода и высокой пропускной способности. Подходит для
больших объемов данных и интенсивных рабочих нагрузок.

7. **Cold HDD (sc1)** — самый дешевый тип EBS томов, предназначенный для данных,
к которым доступ требуется редко. Подходит для архивных данных.

8. **Magnetic Volumes (standard)** — устаревший тип EBS томов на основе магнитных
дисков. Обычно заменяется более производительными типами томов.

9. **I/O Credits (кредиты ввода-вывода)** — система, используемая для измерения
производительности томов gp2. Обеспечивает повышенную производительность в периоды
интенсивной активности.

10. **Burst Performance (всплеск производительности)** — возможность томов gp2
использовать кредиты ввода-вывода для повышения производительности при необходимости.

11. **Volume Attachment (подключение тома)** — процесс подключения тома EBS к
инстансу EC2. Один том может быть подключен только к одному инстансу в определенный
момент времени.

12. **Multi-Attach** — возможность подключения тома EBS к нескольким инстансам
EC2 одновременно. Доступна для томов io1 и io2.

13. **Volume Initialization (инициализация тома)** — процесс подготовки тома
EBS к использованию после создания или восстановления из снимка. Включает
чтение всех блоков данных для обеспечения полной производительности.

14. **Encryption (шифрование)** — функция, обеспечивающая шифрование данных на
томах EBS, включая данные в состоянии покоя, данные при передаче между инстансом
и томом, а также все снимки, созданные из тома. Управляется с помощью
AWS Key Management Service (KMS).

15. **Elastic Volumes (эластичные тома)** — возможность изменения размера,
типа и производительности томов EBS без прерывания работы.

16. **Volume Modifications (модификации тома)** — процесс изменения характеристик
тома EBS, таких как размер, тип и производительность. Модификации можно выполнять
без отключения тома от инстанса.

17. **Fast Snapshot Restore (быстрое восстановление снимков)** — функция, позволяющая
быстро восстанавливать тома EBS из снимков с минимальной задержкой и максимальной
производительностью.

18. **Data Lifecycle Manager (DLM)** — инструмент для автоматического управления
жизненным циклом снимков EBS. Позволяет создавать, сохранять и удалять снимки на
основе заданных правил.

19. **Performance Metrics (метрики производительности)** — показатели, используемые
для мониторинга производительности томов EBS, такие как IOPS, пропускная способность
и задержка. Доступны через Amazon CloudWatch.

20. **Amazon EBS-Optimized Instances (оптимизированные инстансы для EBS)** —
инстансы EC2, которые предоставляют выделенные ресурсы ввода-вывода для томов EBS,
обеспечивая высокую производительность и низкую задержку.
