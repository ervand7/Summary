1. **Amazon EC2 (Elastic Compute Cloud)** — веб-сервис AWS, предоставляющий
масштабируемые вычислительные мощности в облаке. Позволяет запускать виртуальные
серверы, называемые инстансами.

2. **Instance (инстанс)** — виртуальный сервер, запущенный в среде EC2. Инстансы
могут быть настроены с различными типами CPU, памяти, хранилища и сетевых ресурсов.

3. **AMI (Amazon Machine Image)** — шаблон, содержащий программное обеспечение
(операционную систему, приложения и настройки), используемый для запуска инстансов EC2.
Golden Amazon Machine Image (AMI) — это предварительно настроенный образ виртуальной
машины, созданный на базе EC2, который содержит операционную систему, необходимое
программное обеспечение, конфигурации и все зависимости, нужные для запуска вашего
приложения. Это позволяет вам быстро и последовательно развертывать экземпляры EC2 с
уже установленными и настроенными компонентами.

4. **Instance Type (тип инстанса)** — конфигурация инстанса, определяющая
количество CPU, оперативной памяти, хранилища и сетевых возможностей. Примеры
типов инстансов: t2.micro, m5.large, c5.xlarge.

5. **On-Demand Instances (инстансы по запросу)** — тип инстансов, которые
оплачиваются по фиксированной почасовой ставке без долгосрочных обязательств.

6. **Reserved Instances (зарезервированные инстансы)** — инстансы, которые
бронируются на определенный срок (1 или 3 года) за сниженную цену по сравнению
с On-Demand инстансами.

7. **Spot Instances (спотовые инстансы)** — инстансы, которые можно запустить
по сниженной цене, используя неиспользуемые мощности EC2. Цены на спотовые
инстансы варьируются в зависимости от предложения и спроса.

8. **Dedicated Hosts (выделенные хосты)** — физические серверы, полностью
выделенные для вашего использования. Предназначены для соблюдения лицензионных
и нормативных требований.

9. **EIP (Elastic IP Address)** — статический публичный IPv4 адрес, который
можно динамически ассоциировать с любым инстансом в вашем аккаунте.

10. **EBS (Elastic Block Store)** — блочное хранилище для инстансов EC2.
Поддерживает постоянное хранение данных, даже после завершения работы инстанса.

11. **EBS Snapshot (снимок EBS)** — резервная копия тома EBS, которая сохраняется
в Amazon S3. Снимки можно использовать для восстановления томов или создания новых томов.

12. **Instance Store (локальное хранилище инстанса)** — временное хранилище,
предоставляемое некоторыми типами инстансов. Данные на Instance Store теряются
при остановке или завершении инстанса.

13. **Security Group (группа безопасности)** — виртуальный фаервол для инстансов
EC2, который контролирует входящий и исходящий трафик. Правила безопасности можно
применять на уровне инстанса.

14. **Key Pair (пара ключей)** — SSH-ключи, используемые для безопасного подключения
к инстансам EC2. Состоит из открытого и закрытого ключа.

15. **User Data (пользовательские данные)** — скрипты или команды, которые можно
передать инстансу EC2 при его запуске для выполнения при инициализации.

16. **Placement Group (группа размещения)** — логическое объединение инстансов,
обеспечивающее низкую задержку сети или высокую производительность. Типы групп
размещения: Cluster, Spread, Partition.

17. **Auto Scaling (автоматическое масштабирование)** — сервис для автоматического
увеличения или уменьшения количества инстансов EC2 на основе заданных условий.
Обеспечивает масштабируемость и отказоустойчивость приложения.

18. **Load Balancer (балансировщик нагрузки)** — распределяет входящий трафик
между несколькими инстансами EC2, повышая отказоустойчивость и производительность
приложения. AWS предоставляет несколько типов балансировщиков: Classic, Application,
Network.

19. **Launch Template (шаблон запуска)** — шаблон, содержащий конфигурации для
запуска инстансов EC2. Упрощает создание и управление инстансами.

20. **Elastic Network Interface (ENI)** — виртуальный сетевой интерфейс, который
можно прикреплять к инстансам EC2. Поддерживает несколько IP-адресов, групп
безопасности и сетевых настроек.

21. **CloudWatch** — сервис мониторинга для сбора и анализа метрик производительности
инстансов EC2 и других ресурсов AWS. Поддерживает настройку оповещений на основе метрик.

22. **Termination Protection (защита от завершения)** — функция, предотвращающая
случайное завершение инстанса EC2. Может быть включена или отключена для каждого
инстанса.

23. **Elastic Fabric Adapter (EFA)** — сетевой интерфейс, оптимизированный для
приложений с высокими требованиями к пропускной способности и низкой задержке,
таких как высокопроизводительные вычисления (HPC).

24. **Instance Metadata (метаданные инстанса)** — информация о текущем инстансе,
доступная изнутри инстанса по HTTP. Включает в себя данные о конфигурации инстанса,
такие как идентификатор, тип, зона доступности и пользовательские данные.

25. **Spot Fleet** — набор спотовых инстансов, который позволяет запускать и
управлять множеством спотовых инстансов для удовлетворения потребностей в
вычислительных мощностях.
