When to prefer SQL:

1. Complex relationships
* JOINs, aggregations, relational data

2. Strong consistency
* ACID transactions
* Financial and inventory systems

3. Stable schema
* Well-defined data structure
* Strong data integrity

4. Reporting and analytics
* Complex queries over structured data


When to prefer NoSQL:

1. High write throughput
* Social media, events, IoT

2. Flexible schema
* Product catalogs, CMS

3. Large-scale unstructured data
* Logs, posts, sensor data

4. Global distribution
* Multi-region replication
* High availability


Key trade-offs:
* SQL → consistency, transactions, complex queries
* NoSQL → scalability, flexibility, high write throughput

Interview version:
Choose SQL when you need transactions, strong consistency, and complex relationships.
Choose NoSQL when you need horizontal scaling, flexible schemas, or massive write
volumes.
