### Common algorithms

Round robin
* Requests go to instances in order
* Simple, fair for identical instances

Weighted round robin
* More traffic to stronger instances
* Useful with different instance sizes

Least connections
* Sends traffic to the least busy instance
* Good for long or uneven requests

Hash-based routing
* Same key â†’ same instance
* Useful for cache locality or affinity

Consistent hashing
* Minimizes remapping when instances change
* Used in large-scale caching systems
