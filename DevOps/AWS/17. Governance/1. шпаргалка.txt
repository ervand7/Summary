1. **AWS Organizations** — сервис, который позволяет централизованно управлять
и контролировать несколько аккаунтов AWS. Предоставляет возможность создавать
организационные единицы, управлять политиками и отслеживать использование ресурсов.

2. **Service Control Policies (SCP)** — политики управления сервисами, которые применяются
на уровне организации или учетной записи в AWS Organizations. Определяют, какие действия
могут выполняться в аккаунтах организации.

3. **AWS Config** — сервис для оценки, аудита и мониторинга конфигураций ресурсов AWS.
Обеспечивает видимость изменений конфигураций и их соответствие установленным требованиям.

4. **AWS Control Tower** — сервис, который предоставляет упрощенный способ настройки
и управления множественными аккаунтами AWS. Включает в себя предварительно настроенные
политики и настройки безопасности.

5. **AWS CloudTrail** — сервис для мониторинга и логирования активности аккаунта AWS.
Записывает действия, выполняемые пользователями, ролями и сервисами, и предоставляет
историю событий для анализа и аудита.

6. **AWS Trusted Advisor** — инструмент для анализа аккаунта AWS и предоставления
рекомендаций по оптимизации затрат, улучшению производительности, повышению безопасности
и надежности.

7. **AWS Service Catalog** — сервис, который позволяет организациям создавать и управлять
каталогами услуг AWS, одобренных для использования. Обеспечивает контроль и соблюдение
стандартов безопасности и соответствия.

8. **AWS Resource Access Manager (RAM)** — сервис, который позволяет безопасно делиться
ресурсами AWS с другими аккаунтами в организации или за ее пределами.

9. **AWS Audit Manager** — сервис, который помогает автоматизировать процессы аудита для
упрощения проверки соответствия стандартам и нормативным требованиям.

10. **AWS License Manager** — сервис, который помогает управлять лицензиями на программное
обеспечение, используемое в AWS и в локальной инфраструктуре. Обеспечивает контроль
использования лицензий и соблюдение условий лицензионных соглашений.

11. **AWS Well-Architected Tool** — инструмент, который помогает проверять рабочие нагрузки
на соответствие лучшим практикам AWS по пяти основным направлениям: операционная
эффективность, безопасность, надежность, производительность и оптимизация затрат.

12. **Tagging (тегирование)** — процесс присвоения меток (тегов) ресурсам AWS для упрощения
их организации, управления и отслеживания затрат. Теги состоят из ключа и значения.

13. **Cost Allocation Tags (теги распределения затрат)** — теги, используемые для
распределения и отслеживания затрат по различным проектам, отделам или другим категориям.

14. **AWS Budgets** — сервис для установки и отслеживания бюджетов на использование и
затраты AWS. Позволяет получать уведомления при превышении установленных лимитов.

15. **AWS Cost Explorer** — инструмент для анализа затрат на использование AWS.
Предоставляет визуализацию данных о расходах и помогает выявлять тенденции и аномалии.

16. **AWS Compute Optimizer** — сервис, который помогает оптимизировать использование
ресурсов AWS, таких как инстансы EC2, на основе анализа их производительности и использования.
Предоставляет рекомендации по выбору более подходящих типов инстансов для улучшения
производительности и снижения затрат.
