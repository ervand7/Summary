Lambda concurrency = how many executions run at the same time.

Account concurrency (default limit):

* Account-wide limit (e.g. 1,000 concurrent executions per region)
* Shared by all Lambda functions in the account
* Exceeding it causes throttling

Reserved concurrency (per function):

* Hard limit for a specific Lambda function
* Guarantees capacity for this function
* Also caps it so it cannot exceed this number
* Protects critical Lambdas from being starved by others

Example:
* Account limit = 1,000
* Function A reserved = 200 â†’ always has up to 200 available
* Remaining pool = 800 for other functions

Provisioned concurrency (warm capacity):

* Pre-initialized execution environments
* Eliminates cold starts
* You pay even when idle
* Scales instantly up to the provisioned value

Key differences:

* Reserved concurrency â†’ capacity isolation + throttling control
* Provisioned concurrency â†’ performance (no cold starts)
* Account limit â†’ global ceiling
