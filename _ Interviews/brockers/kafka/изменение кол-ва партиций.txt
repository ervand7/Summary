The number of partitions in Kafka can only be increased, not decreased,
because reducing partitions would break message ordering and key-based distribution.

Who increases partitions:
Typically an administrator (DevOps), or less commonly the application via the Kafka Admin API.
Kafka never does this automatically.

When to increase:
When throughput is insufficient, load grows, or more consumer parallelism is needed.

How to increase:
Via an explicit command or API, for example:

kafka-topics.sh --alter --topic my-topic --partitions 12

This is a manual operation that can be performed at runtime;
after that, the consumer group will automatically rebalance.
