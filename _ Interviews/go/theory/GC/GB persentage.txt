1) GOGC
Environment variable that controls GC frequency.
GOGC=100 → GC runs when heap grows by 100%.
Lower = more frequent GC, higher = less frequent.
(soft threshold)

2) Pacer
Controls *when and how fast* GC runs
→ keeps balance between memory cleanup and app performance

3) GC Percentage (GP)
% of time spent in GC vs program work
High value → GC takes significant time
