asyncio.gather:
* Runs coroutines concurrently
* Returns results in order
* Fails fast on first exception (by default)

asyncio.wait:
* Waits for futures to complete
* Returns done and pending sets
* Gives more control over completion behavior

Rule of thumb:
> gather for results, wait for control

## `asyncio.wait` â€” *â€œtell me who finishedâ€*

### Example

```python
import asyncio

async def job(n):
    await asyncio.sleep(n)
    return n

async def main():
    tasks = [
        asyncio.create_task(job(1)),
        asyncio.create_task(job(2)),
    ]

    done, pending = await asyncio.wait(tasks)
    print(len(done), len(pending))

asyncio.run(main())
```

Output

```
2 0
```

### What it does

* Waits for tasks to complete
* Returns two sets: `done`, `pending`
* No results automatically
* You decide what to do next

ðŸ‘‰ Use when you need control

---

## Early return example (`wait` advantage)

```python
done, pending = await asyncio.wait(
    tasks,
    return_when=asyncio.FIRST_COMPLETED
)
```

* Returns when first task finishes
* Others stay pending
