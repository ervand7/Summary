Переменные окружения живут только в одной сессии терминала.
Откроем терминал:
 ● export TEST_ENV=8
 ● echo TEST_ENV
TEST_ENV
 ● echo $TEST_ENV
8

 ● python3
 ● import os
 ● os.getenv('TEST_ENV')
8
 ● TOKEN = os.getenv('TEST_ENV')
Закроем терминал

Откроем терминал:
 ● echo $TEST_ENV
null

___________________________________________________________________
Создадим переменную для конкретной программы
 ● MY_ENV=10 python3
 ● python3
 ● import os
 ● os.getenv('MY_ENV')
10
Важно! Сейчас мы создали переменную окружения для питоновской программы, а не для терминала.
Так что, если бы у нас была одноименная переменная в терминале, то после закрытия питонвской
программы, у нас эта переменная принимала бы значение, которе мы ей когда-то задали в текущей
сессии терминала.
