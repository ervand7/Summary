1. Producer → Exchange

* Producers always publish to exchanges, never directly to queues.
* One producer can publish to multiple exchanges; many producers can share an exchange.

2. Exchange → Queue (bindings)

* Exchanges route messages to one or multiple queues via bindings.
* A queue can receive messages from multiple exchanges.

3. Queue → Consumer

* Queues can be consumed by one or many consumers (competing consumers).
* Each message is delivered to only one consumer per queue.

4. Delivery & Ordering

* Exchanges can fan out messages to multiple queues; each queue holds its own copy.
* Message priority is set by the producer; queues determine delivery order.
* Consumers just read the queue’s ready messages.

Key Takeaway:

> Producers send messages to exchanges → exchanges route to queues → queues are consumed
by one or more consumers. Each message is handled by a single consumer per queue.
