**CORS (Cross-Origin Resource Sharing)** — это механизм безопасности, который позволяет
веб-серверам управлять доступом к своим ресурсам с других источников (доменов). По умолчанию
браузеры блокируют запросы, которые исходят с одного домена и направлены на другой, чтобы
защитить пользователей от потенциальных угроз, таких как **межсайтовый запрос данных (CSRF)**.

CORS позволяет веб-серверу явно указывать, какие домены имеют право взаимодействовать с его
ресурсами. Это достигается с помощью специальных HTTP-заголовков, которые сервер отправляет в
ответе на запрос:

- **Access-Control-Allow-Origin**: Указывает домен, который может получить доступ к ресурсу.
Например, `Access-Control-Allow-Origin: https://example.com` позволяет только домену `example.com`
доступ к ресурсу. Можно также установить `*`, чтобы разрешить доступ с любых источников, но
это не всегда безопасно.

- **Access-Control-Allow-Methods**: Указывает, какие HTTP-методы (GET, POST, PUT и т. д.)
разрешены для данного ресурса.

- **Access-Control-Allow-Headers**: Указывает, какие заголовки можно использовать в запросах с
других источников.

- **Access-Control-Allow-Credentials**: Определяет, разрешены ли кросс-доменные запросы с
учётными данными (такими как куки и заголовки авторизации).

### Как работает CORS
Когда браузер делает кросс-доменный запрос (например, когда веб-страница на `example.com`
отправляет запрос к API на `api.example.org`), браузер сначала отправляет так называемый
**предварительный запрос (preflight request)**, чтобы проверить, разрешены ли запросы на
целевом сервере. Этот запрос использует метод OPTIONS и проверяет, какие методы и заголовки
поддерживает сервер. Если сервер отвечает положительно, браузер выполняет фактический запрос.

### Зачем нужен CORS
CORS нужен для обеспечения безопасности, чтобы веб-приложения могли получать доступ к ресурсам
на других доменах только с разрешения. Это предотвращает вредоносные сайты от
несанкционированного доступа к данным, к примеру, из API вашего сайта, и защищает
пользователей от атак, связанных с передачей данных между разными источниками.
