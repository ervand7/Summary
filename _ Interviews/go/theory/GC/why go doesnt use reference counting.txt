Reference counting — это подход, где у каждого объекта есть счётчик ссылок.
- +1 при создании ссылки
- -1 при удалении
- если = 0 → объект удаляется


## ❌ Почему Go не использует reference counting
1. Не удаляет циклы
a -> b -> a

Счётчики не равны 0 → утечка памяти
2. Дорого на каждое присваивание
   Каждое `x = y` → инкремент/декремент счётчиков
   → замедляет программу

3. Плохо для конкурентности
   Сложно сделать потокобезопасным и быстрым для горутин


## ✅ Почему Go использует Mark & Sweep
* Удаляет циклы
* Работает параллельно (concurrent GC)
* Нет накладных расходов на каждую операцию


## 🧠 Коротко:
* Reference counting → просто, но слабое (проблемы с циклами + overhead)
* Go GC (tracing) → сложнее, но мощнее и эффективнее
