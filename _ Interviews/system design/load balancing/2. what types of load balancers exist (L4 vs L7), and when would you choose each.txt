L4 and L7 load balancers work at different layers.

L4 (Transport):
* Balances TCP connections
* Sees only IP/port
* Very fast
* Good for gRPC, TCP, TLS passthrough

L7 (Application):
* Balances HTTP requests
* Sees paths, headers, methods
* Supports routing, auth, rate limits
* Used in API gateways and microservices

Typical production setup:
Client → L4 → L7 proxy → Apps

Key insight:
* L4 distributes connections
* L7 distributes requests

Interview version:
L4 load balancers route TCP connections, while L7 load balancers route HTTP requests
with application-level logic; production systems often use both together.
