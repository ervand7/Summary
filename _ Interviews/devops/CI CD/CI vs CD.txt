Continuous Integration (CI)
Developers frequently merge code into a shared repository, and every commit triggers
automatic builds and tests to catch errors early.

Continuous Delivery (CD)
After CI passes, the system automatically prepares the application for release.
Deployment to production is a manual decision.

Continuous Deployment
Like Continuous Delivery, but deployment to production happens automatically after
all tests pass.

---

### Simple flow
commit → CI (build + tests) → CD (release-ready) → deploy

---

### Quick comparison

| Concept                   | Purpose                                           |
| ------------------------- | ------------------------------------------------- |
| CI                        | Automatically build and test code on every commit |
| Continuous Delivery       | Keep software always ready for release            |
| Continuous Deployment     | Automatically release to production               |

In short:
* CI = integrate and test code
* CD = deliver/deploy the software.
