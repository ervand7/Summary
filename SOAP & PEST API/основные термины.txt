REST и SOAP на самом деле не сопоставимы. REST — это архитектурный стиль. SOAP — это формат обмена сообщениями.

 ● REST - это архитектурный стиль
 Системы, поддерживающие REST, называются RESTful-системами.
 Особенности:
1) Нет стандарта
2) Мягкий: возможны xml, json и др.
3) WADL (Web Application Description Language) по желанию разработчика. В основном, только в Enterprise
4) Может работать только по HTTP-протоколу
5) Используется в широком вебе

 ● SOAP (от англ. Simple Object Access Protocol — простой протокол доступа к объектам) — протокол обмена
структурированными сообщениями в распределённой вычислительной среде.
Особенности:
1) SOAP - это стандарт
2) Жесткий: только XML
3) Всегда есть схема WSDL со всеми параметрами
4) Может работать по многим протоколам (HTTP, SMTP, FTP)
5) Используется только в Enterprise. Так как тут много метаинформации, которая сильно утежеляла бы веб

 ● WADL - машинно-читаемое XML-описание для web-приложений HTTP (как правило, веб-сервисы REST).
Аналог WSDL для SOAP.

 ● WSDL (Web Services Description Language) - описание всех методов, которые нам доступны через SOAP. Это язык описания
веб-сервисов и доступа к ним, основанный на языке XML.
Вот пример WSDL: http://users.bugred.ru/tasks/soap/WrapperSoapServer.php?wsdl

 ● Заголовки выступают в роли маршрутизатора и уже указывают, является ли HTTP-сообщение SOAP или REST.
Эти заголовки автоматически генерируются программами SOAP UI (SOAP) и Postman (REST)
