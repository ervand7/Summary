| Критерий            | **SSE (Server-Sent Events)**  | **WebSockets**                       |
| ------------------- | ----------------------------- | ------------------------------------ |
| Направление         | **Только сервер → клиент**    | **Двустороннее**                     |
| Тип соединения      | HTTP (long-lived)             | Отдельный WS-протокол                |
| Инициатор           | Клиент открывает, сервер шлёт | Клиент открывает, дальше обе стороны |
| Формат              | Текст (`text/event-stream`)   | Любой (text/binary)                  |
| Поддержка браузеров | Отличная (кроме старых IE)    | Отличная                             |
| Автопереподключение | **Есть из коробки**           | Нужно реализовывать                  |
| Backpressure        | Простой (TCP)                 | Нужно проектировать                  |
| Прокси / LB         | **Хорошо дружит с HTTP**      | Часто требует настройки              |
| Масштабирование     | Проще                         | Сложнее                              |
| Типичные кейсы      | Уведомления, прогресс, ленты  | Чаты, игры, коллаборация             |
| Latency             | Низкая                        | Очень низкая                         |
| Server complexity   | Низкая                        | Выше                                 |

---

### Когда что выбирать (1-лайнеры)

* **SSE** →
  *«Сервер шлёт обновления, клиент только слушает»*
  Примеры: нотификации, лог-стриминг, live-metrics, news feed

* **WebSockets** →
  *«Нужен real-time диалог»*
  Примеры: чат, онлайн-игры, совместное редактирование
