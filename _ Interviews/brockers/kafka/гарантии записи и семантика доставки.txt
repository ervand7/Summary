## 1. `acks` — write guarantee

`acks` defines when the partition leader acknowledges a produce request:

* `acks=0`  — no acknowledgment (fast, may lose data)
* `acks=1`  — leader acknowledges after local write
* `acks=all` — leader waits for all in-sync replicas (ISR) to replicate


## 2. Delivery semantics

at-most-once / at-least-once / exactly-once are end-to-end guarantees,
not a single Kafka setting.

They depend on:
* producer config
* when consumer commits offsets
* idempotency and transactions


## 3. Semantics

At-most-once
* commit BEFORE processing
* possible data loss, no duplicates

At-least-once (default)
* commit AFTER processing
* no loss, duplicates possible

Exactly-once
* requires idempotent producer + transactions
* commit offsets within transaction
