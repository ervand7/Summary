DB connection pool size is a concurrency limit for database access.

* 1 DB connection ≈ 1 query running in parallel
* Pool size = max number of concurrent DB operations
* Extra requests wait in the application queue

Formula:
DB parallelism ≤ pool size

Why it matters:
* protects the database from overload
* creates backpressure
* keeps latency more stable
* prevents timeout and retry storms

How to choose it:
* start from the DB capacity, not from RPS
* divide safe DB connections across all app instances
* start small, measure, then increase carefully

Example:
3 app instances × pool size 4 = 12 total DB connections

Interview version:
A DB connection pool limits how many queries can hit the database concurrently,
trading some queuing in the app for system stability.
