1. **Amazon S3 (Simple Storage Service)** — объектное хранилище данных,
предоставляемое AWS, позволяющее хранить и извлекать любые объемы данных в любое время.

2. **Bucket (бакет)** — контейнер для хранения объектов (файлов) в S3. Все объекты
хранятся в бакетах, которые имеют уникальные имена.

3. **Object (объект)** — основной элемент хранения данных в S3. Состоит из данных
(файла) и метаданных (дополнительной информации об объекте).

4. **Key (ключ)** — уникальный идентификатор объекта внутри бакета. Вместе с
именем бакета ключ используется для доступа к объекту.

5. **Versioning (версионность)** — функция, позволяющая сохранять несколько версий
одного объекта в бакете. Полезна для защиты данных от случайного удаления или перезаписи.

6. **Lifecycle Policy (политика жизненного цикла)** — правила, определяющие
автоматическое перемещение или удаление объектов на основе их возраста. Используется
для управления стоимостью хранения.

7. **Storage Class (класс хранения)** — определяет уровень доступности и стоимость
хранения данных. Основные классы хранения включают
S3 Standard
S3 Intelligent-Tiering
S3 Standard-IA
S3 One Zone-IA
S3 Glacier
S3 Glacier Deep Archive.

8. **S3 Standard** — класс хранения с высокой доступностью и низкой задержкой,
предназначенный для часто используемых данных.

9. **S3 Intelligent-Tiering** — класс хранения, автоматически перемещающий
данные между двумя уровнями доступа (частый и редкий) на основе изменения
паттернов доступа.

10. **S3 Standard-IA (Infrequent Access)** — класс хранения для данных, к
которым доступ требуется реже, но которые должны быть доступны немедленно при
необходимости.

11. **S3 One Zone-IA** — класс хранения для данных, к которым доступ требуется
редко, и которые хранятся в одной зоне доступности (AZ). Более дешевый
вариант по сравнению с S3 Standard-IA.

12. **S3 Glacier** — класс хранения для архивных данных, к которым доступ
требуется нечасто. Подходит для данных, которые могут ждать восстановления
от нескольких минут до часов.

13. **S3 Glacier Deep Archive** — самый дешевый класс хранения для архивных данных,
к которым доступ требуется очень редко (например, раз в год). Время восстановления
данных составляет от 12 до 48 часов.

14. **Bucket Policy (политика бакета)** — набор разрешений на уровне бакета,
определяющий, кто и какие действия может выполнять с объектами в бакете.

15. **Access Control List (ACL)** — список управления доступом, определяющий
права доступа к объектам и бакетам на уровне пользователя или группы.

16. **Pre-Signed URL (предварительно подписанный URL)** — временный URL, который
позволяет ограниченному доступу к объектам в S3. Может быть использован для
предоставления временного доступа к приватным объектам.

17. **Server-Side Encryption (серверное шифрование)** — шифрование данных на
стороне сервера. S3 поддерживает несколько вариантов шифрования:
SSE-S3 (управляется S3)
SSE-KMS (управляется KMS)
SSE-C (управляется клиентом).

18. **Client-Side Encryption (шифрование на стороне клиента)** — шифрование
данных перед их загрузкой в S3. Расшифровка происходит на стороне клиента после
загрузки данных из S3.

19. **Cross-Region Replication (CRR)** — функция, позволяющая автоматически
реплицировать объекты из одного бакета в другой бакет в другом регионе AWS для
повышения доступности и устойчивости данных.

20. **Transfer Acceleration** — функция, использующая глобальную сеть AWS для
ускорения загрузки и скачивания данных из S3.
This feature can significantly speed up data transfers over long distances, which is
ideal for an international company with clients around the world.

21. **Multipart Upload (многократная загрузка)** — метод загрузки больших объектов
в S3 частями. Улучшает надежность и скорость загрузки.

22. **Event Notifications (уведомления о событиях)** — функция, позволяющая
настроить уведомления при различных событиях в бакете, таких как создание
нового объекта, удаление объекта и т.д. Уведомления могут отправляться в
AWS Lambda, SQS или SNS.

23. **Data Consistency (консистентность данных)** — S3 предоставляет сильную
консистентность данных для всех операций загрузки и перезаписи объектов.

24. **Static Website Hosting (хостинг статического веб-сайта)** — возможность
хостинга статических веб-сайтов напрямую из S3 бакета.
