* В один Kafka-топик могут писать сколько угодно продюсеров одновременно
* Продюсеры не блокируют друг друга, глобального порядка сообщений нет
* Kafka гарантирует порядок только внутри одной partition
* Если продюсеры отправляют сообщения с одинаковым key (например `user_1`)
  → Kafka по `hash(key)` отправляет их в одну и ту же partition
* В результате:
  * все события `user_1` попадают в одну partition
  * их порядок строго сохраняется
* Гарантия нарушается, если:
  * изменили число partition
  * используется кастомный partitioner
  * сообщения отправляются без key

Формулировка для интервью:
> Kafka обеспечивает упорядоченность сообщений по ключу внутри одной partition,
даже при наличии нескольких продюсеров.
