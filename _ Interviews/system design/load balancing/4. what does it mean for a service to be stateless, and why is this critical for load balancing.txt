A stateless service stores no per-user state in memory between requests.

Why it matters:
* Any request can go to any instance
* No sticky sessions are needed
* Instances can scale horizontally easily
* Crashed instances lose no user state

What breaks statelessness:
* In-memory sessions
* User-specific local caches
* Request affinity

Interview version:
A stateless service keeps no user state in memory, allowing any request to be
handled by any instance, which simplifies load balancing and scaling.
