What is the maximum number of messages that can be processed in parallel:
given 1 topic, 3 partitions, and 6 consumers in the same consumer group?

Correct answer:
- 3

Explanation:
In Kafka, each message belongs to a partition. Within a consumer group, each partition
can be assigned to only one consumer. Therefore, even with 6 consumers and only 3 partitions,
only 3 messages can be processed in parallel — one per partition.

Parallelism is determined by the number of partitions, not the number of consumers.
