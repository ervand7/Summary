Load balancing distributes traffic across multiple service instances to prevent
overload and improve availability.

Why it matters:
* One server has CPU, memory, and connection limits
* Overloaded instances become slow and unstable
* Multiple instances allow horizontal scaling
* Failed instances can be removed automatically

Benefits:
* Better scalability
* Higher availability
* Fault isolation
* Graceful degradation under load

Key Go insight:
* Goroutines help concurrency, but a single instance still hits resource limits

Interview version:
Load balancing spreads traffic across multiple instances to prevent overload,
enable horizontal scaling, and improve reliability.
