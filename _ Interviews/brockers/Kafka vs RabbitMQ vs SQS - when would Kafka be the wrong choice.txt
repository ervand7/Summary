Here’s a concise comparison:

| Feature / Use Case     | **Kafka**                       | **RabbitMQ**                   | **SQS**                       |
| ---------------------- | ------------------------------- | ------------------------------ | ----------------------------- |
| **Purpose**            | High-throughput event streaming | Flexible messaging & routing   | Simple, managed message queue |
| **Latency**            | Milliseconds–hundreds ms        | Low (ms)                       | Moderate (~tens ms)           |
| **Throughput**         | Very high                       | Moderate                       | Moderate                      |
| **Message ordering**   | Per partition                   | Guaranteed per queue           | FIFO optional                 |
| **Routing & patterns** | Minimal (topic/partition)       | Advanced (exchanges, bindings) | Basic                         |
| **Replay / retention** | Yes (log, time travel)          | No                             | No                            |
| **Ops / management**   | More effort                     | Moderate                       | Almost zero                   |
| **Best for**           | Event streams, analytics        | Complex routing, task queues   | Simple queue, serverless apps |


> Kafka = event log, RabbitMQ = flexible queue, SQS = managed simple queue.
