Стандартная утилита godoc генерирует документацию для Go-проекта на
основе комментариев, указанных в исходном коде. При этом godoc работает
как веб-сервер и позволяет просматривать документацию в браузере —
можно кликнуть по любой функции и перейти к её исходному коду.
Для установки godoc нужно запустить команду:
 ● go install -v golang.org/x/tools/cmd/godoc@latest

Чтобы документация пакета или модуля отображалась корректно, разработчику
нужно правильно оформить его описание:
 - Общие комментарии лучше поставить перед объявлением имени пакета.
 - В первом предложении стоит указать основное назначение пакета. Это
 предложение будет отображаться в общем списке пакетов.
 - Нужно следить за тем, чтобы все предложения в документации были законченными
 по смыслу и оканчивались точкой.
 - Если описание занимает много места или пакет состоит из нескольких .go-файлов,
 нужно создать отдельный файл doc.go, в котором будет информация о пакете.
 Например, вводная часть https://pkg.go.dev/net/http документации к пакету
 net/http взята из файла doc.go. https://cs.opensource.google/go/go/+/master:src/net/http/doc.go

Рассмотрим правила оформления документации на примере модуля retrier, который
позволяет повторять операцию до её успешного завершения с изменяющимися
интервалами между попытками.

 ● godoc -http=:8080
нужно запускать из директории retrier
По умолчанию godoc не отображает пакеты, расположенные в поддиректориях internal.
Чтобы увидеть служебные пакеты, добавьте в браузере
параметр ?m=all: например, http://localhost:8080/pkg/?m=all.

Для локального отображения godoc-документации можно воспользоваться утилитами:
 ● godoc -http=:8080
 - godoc запускает локальный веб-сервис (аналогичный pkg.go.dev) на порте 8080.
Можно указать другой доступный порт.
 - По умолчанию сервис отображает документацию по всем пакетам в $GOROOT/src (пакеты
стандартной библиотеки) и документацию пакета в текущей директории, если
там есть файл go.mod.
 - Чтобы скачать документацию в формате HTML, можно использовать wget. Например, wget -r -np -N -E -p -k http://localhost:8080/pkg/github.com/user/retrier/.
 ● go doc -all ./retrier
go doc выводит документацию для указанного модуля (или конкретной функции модуля) в консоль.
Вот пример вызова для функции fmt.Printf стандартной библиотеки:
 ● go doc fmt Printf.

Возможно, вы заметили, что с помощью godoc -http=:8080 невозможно запустить
все примеры стандартной библиотеки. Чтобы примеры кода были запускаемыми, используйте
параметр -play:
 ● godoc -http=:8080 -play.
