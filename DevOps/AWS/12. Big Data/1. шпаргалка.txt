1. **Amazon Kinesis** — семейство сервисов для обработки и анализа потоковых
данных в реальном времени.

2. **Amazon Kinesis Data Streams (KDS)** — сервис для захвата, хранения и обработки
потоков данных в реальном времени. Подходит для обработки большого объема данных
с минимальной задержкой.

3. **Amazon Kinesis Data Firehose** — полностью управляемый сервис для захвата,
преобразования и загрузки потоковых данных в хранилища данных, такие как Amazon S3,
Amazon Redshift, Amazon Elasticsearch Service (OpenSearch Service), и Splunk.

4. **Amazon Kinesis Data Analytics** — сервис для анализа потоковых данных с
использованием стандартного SQL. Позволяет извлекать ценные инсайты из данных
в реальном времени, поступающих через Kinesis Data Streams или Kinesis Data Firehose.

5. **Shard (шард)** — базовый блок данных в Kinesis Data Streams. Каждый поток
(stream) состоит из одного или нескольких шардов, и каждый шард обеспечивает
фиксированную пропускную способность.

6. **Producer (производитель)** — приложение или сервис, который отправляет
данные в поток Kinesis Data Streams.

7. **Consumer (потребитель)** — приложение или сервис, который читает и обрабатывает
данные из потока Kinesis Data Streams. Примеры потребителей: AWS Lambda,
Kinesis Client Library (KCL).

8. **Amazon Kinesis Video Streams** — сервис для захвата, хранения и обработки
потокового видео и аудио.

9. **Record (запись)** — единица данных, отправляемая в поток Kinesis Data Streams.
Состоит из ключа партиционирования, данных и метаданных.

10. **Partition Key (ключ партиционирования)** — ключ, используемый для распределения
записей по шардам в потоке Kinesis Data Streams.

11. **Kinesis Agent** — агент для автоматической отправки журналов (логов) и других
файловых данных в Kinesis Data Streams или Kinesis Data Firehose.

12. **AWS Lambda** — серверлесс-сервис, который может выступать в роли потребителя
данных из Kinesis Data Streams, выполняя обработку данных в реальном времени.

13. **Amazon CloudWatch** — сервис для мониторинга и управления ресурсами AWS.
Используется для мониторинга метрик Kinesis, таких как задержка, пропускная
способность и использование шардов.

14. **DynamoDB Streams** — функция Amazon DynamoDB, которая захватывает изменения
в таблицах DynamoDB и отправляет их в поток. Эти потоки могут быть интегрированы
с Kinesis Data Streams для дальнейшей обработки.

15. **Elastic Load Balancer (ELB)** — распределяет входящий трафик между несколькими
целевыми ресурсами, такими как EC2 инстансы. Может использоваться в сочетании с
Kinesis для обработки потоков данных от множества источников.

16. **Kinesis Client Library (KCL)** — библиотека, упрощающая написание
приложений-потребителей, которые обрабатывают данные из Kinesis Data Streams.

17. **AWS Glue** — это полностью управляемый сервис от Amazon для извлечения, преобразования
и загрузки данных (ETL). Он помогает автоматизировать процесс подготовки данных для анализа.
Основные функции AWS Glue:
- **Извлечение данных** из разных источников.
- **Преобразование данных** в нужный формат.
- **Загрузка данных** в хранилища данных, такие как Amazon S3 или Amazon Redshift.
AWS Glue автоматически обнаруживает и каталогизирует данные, создавая метаданные,
чтобы упростить их дальнейшее использование.
