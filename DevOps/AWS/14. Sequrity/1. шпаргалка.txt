1. **Identity Federation (федерация идентичности)** — возможность использовать
внешние системы управления доступом (например, Active Directory) для
аутентификации и авторизации в AWS.

2. **Resource-Based Policies (политики на основе ресурсов)** — политики,
которые можно прикрепить непосредственно к ресурсам, таким как S3-бакеты или
SNS-топики. Эти политики определяют, кто и какие действия может выполнять с ресурсом.

3. **IAM Permissions Boundary (граница разрешений IAM)** — функция, позволяющая
ограничить максимальные привилегии, которые могут быть назначены пользователю
или роли.

4. **Service Control Policies (SCP)** — политики, используемые AWS Organizations
для управления разрешениями на уровне организации или учетной записи. Эти политики
определяют, какие действия могут выполняться в аккаунтах организации.

5. **AWS KMS (Key Management Service)** — сервис для управления криптографическими
ключами, используемыми для шифрования данных. Поддерживает создание, управление
и использование ключей шифрования.

6. **AWS CloudTrail** — сервис для мониторинга и логирования активности аккаунта
AWS. Записывает действия, выполняемые пользователями, ролями и сервисами, и
предоставляет историю событий для анализа и аудита.

7. **AWS Config** — сервис для оценки, аудита и мониторинга конфигураций
ресурсов AWS. Обеспечивает видимость изменений конфигураций и их соответствие
установленным требованиям.

8. **AWS Shield** — управляемый сервис защиты от DDoS атак. Shield Standard
предоставляется бесплатно и защищает ресурсы AWS от большинства распространенных
атак. Shield Advanced предлагает дополнительную защиту и поддержку.

9. **AWS WAF (Web Application Firewall)** — веб-аппликационный фаервол,
который защищает веб-приложения от распространенных веб-угроз, таких как
SQL-инъекции и скриптинг межсайтовых запросов (XSS).

10. **AWS Inspector** — автоматизированный сервис для оценки безопасности
приложений, развернутых на инстансах EC2. Проверяет уязвимости и соблюдение
лучших практик безопасности.

11. **AWS Macie** — сервис для автоматического обнаружения, классификации
и защиты чувствительных данных, таких как персональные данные (PII) и
интеллектуальная собственность.

12. **AWS GuardDuty** — управляемый сервис для обнаружения угроз, который
постоянно мониторит ваши учетные записи и данные на предмет вредоносной
активности и несанкционированного поведения.

13. **AWS Secrets Manager** — сервис для управления секретами, такими как
учетные данные базы данных, API ключи и другие чувствительные данные.
Поддерживает автоматическое обновление и ротацию секретов.

14. **AWS Certificate Manager (ACM)** — сервис для управления SSL/TLS
сертификатами. Обеспечивает простое развертывание, управление и ротацию
сертификатов для использования с ресурсами AWS и собственными доменами.

15. **Security Groups (группы безопасности)** — виртуальные фаерволы,
которые контролируют входящий и исходящий трафик для ресурсов AWS, таких
как инстансы EC2.

16. **Network ACLs (сетевые ACL)** — контрольные списки доступа на уровне
подсети, которые контролируют входящий и исходящий трафик. Применяются ко
всем инстансам в подсети.

17. **Encryption in Transit (шифрование при передаче данных)** — метод защиты
данных при их передаче между клиентами и серверами или между сервисами.

18. **Encryption at Rest (шифрование данных в состоянии покоя)** — метод
защиты данных, когда они хранятся на дисках, в базах данных или других хранилищах.

19. **VPC Peering** — связь между двумя VPC, которая позволяет им обмениваться
трафиком, как будто они находятся в одной сети, без выхода в интернет.

20. **AWS Security Hub** — централизованная панель для управления безопасностью,
которая собирает, анализирует и визуализирует данные безопасности из различных
сервисов AWS и партнерских решений.
