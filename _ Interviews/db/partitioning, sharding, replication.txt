## Схема №1 — ОДНА БД (НЕ шардирование)
```
[ Server 1 ]
└── Database A
    └── users
        ├── id 1..1M
        ├── id 1M..2M
        └── id 2M..3M
```

* Один сервер
* Одна БД
* Одна таблица
* Даже если есть партиции — это не шардирование


## Схема №2 — РЕПЛИКАЦИЯ (несколько серверов, но всё ещё НЕ шардирование)
```
[ Server 1 ]        [ Server 2 ]        [ Server 3 ]
└── Database A  ──▶ └── Database A  ──▶ └── Database A
    └── users          └── users          └── users
       (1..3M)            (1..3M)            (1..3M)
```

* Несколько серверов
* Одна и та же БД целиком
* Все данные есть на каждом сервере
  ➡ Это репликация


## Схема №3 — ШАРДИРОВАНИЕ (вот ключевой момент)
```
[ Server 1 ]        [ Server 2 ]        [ Server 3 ]
└── Database A      └── Database B      └── Database C
    └── users           └── users           └── users
       (1..1M)             (1M..2M)           (2M..3M)
```

❗ Обрати внимание:
* РАЗНЫЕ базы данных
* Нет сервера с полными данными
* Таблица `users` — логически одна,
  но физически разорвана

➡ Это шардирование

---


## Итог одной фразой:
> При шардировании есть одна логическая таблица,
> но несколько физических баз данных,
> каждая из которых хранит только часть данных.
