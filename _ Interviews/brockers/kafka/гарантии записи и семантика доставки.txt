## 1. `acks` — write guarantee
`acks` is a producer setting that defines when Kafka considers a message successfully written.

It determines:
how many replicas (brokers) in the ISR must confirm the write to the partition log
before the producer receives an acknowledgment.

Values:
* `acks=0` — no acknowledgments (fast, but messages can be lost)
* `acks=1` — only the leader acknowledges
* `acks=all` — all ISR replicas acknowledge


## 2. at-most-once / at-least-once / exactly-once — delivery semantics
These are not a single Kafka setting, but end-to-end system behavior.

In short:
> Delivery semantics describe how the system delivers and processes messages.

They depend on:
* producer logic
* when offsets are committed in the consumer
* idempotency
* transactions
