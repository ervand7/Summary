1. **Amazon RDS (Relational Database Service)** — управляемый сервис реляционных
баз данных, упрощающий настройку, эксплуатацию и масштабирование реляционных баз
данных в облаке. Поддерживает MySQL, PostgreSQL, MariaDB, Oracle, и SQL Server.

2. **Amazon Aurora** — управляемая реляционная база данных, совместимая с MySQL и
PostgreSQL. Обеспечивает высокую производительность и доступность, а также автоматическое
масштабирование и резервное копирование.

3. **Multi-AZ Deployment** — конфигурация для Amazon RDS и Amazon Aurora, обеспечивающая
повышенную доступность и отказоустойчивость базы данных. Данные автоматически реплицируются
на резервную базу данных в другой зоне доступности (AZ).

4. **Read Replica** — копия базы данных, используемая для распределения нагрузки на чтение
и повышения производительности приложений. Read Replica могут находиться в том же регионе
или в другом регионе.

5. **Amazon DynamoDB** — полностью управляемая NoSQL база данных, обеспечивающая быструю и
предсказуемую производительность при масштабировании. Поддерживает документный и ключ-значение
модели данных.
DynamoDB auto-scaling allows your table's read and write capacity to automatically
adjust based on the incoming traffic. It can automatically increase provisioned capacity
to accommodate higher request rates and reduce capacity during lower traffic periods.

6. **DynamoDB Streams** — функция DynamoDB, которая захватывает изменения в таблицах DynamoDB
и отправляет их в поток для дальнейшей обработки.

7. **Amazon Redshift** — управляемый сервис для аналитики и хранилища данных, предназначенный
для выполнения сложных запросов на больших объемах данных. Поддерживает колоночное хранилище
данных и параллельную обработку запросов.

8. **Amazon ElastiCache** — управляемый сервис для разворачивания и управления кэшами памяти,
таких как Redis и Memcached, для ускорения приложений.

9. **Amazon Neptune** — управляемая графовая база данных, поддерживающая модели графов Property
Graph и RDF (Resource Description Framework).

10. **Amazon DocumentDB** — полностью управляемая документная база данных, совместимая с
MongoDB, предназначенная для хранения и управления JSON-документами.

11. **Amazon QLDB (Quantum Ledger Database)** — полностью управляемая база данных,
предоставляющая неизменяемый журнал транзакций, контролируемый централизованно. Позволяет
вести полный и проверяемый журнал транзакций.

12. **Amazon Keyspaces (for Apache Cassandra)** — полностью управляемый сервис, совместимый
с Apache Cassandra, для управления масштабируемыми базами данных с низкой задержкой.

13. **Amazon Timestream** — полностью управляемая временная база данных для интернета вещей
(IoT) и операционных приложений, предназначенная для хранения и анализа временных данных.

14. **Database Migration Service (DMS)** — сервис для миграции баз данных в AWS с минимальными
простоями. Поддерживает гомогенные и гетерогенные миграции.

15. **Schema Conversion Tool (SCT)** — инструмент для автоматической конвертации схемы базы
данных и кода приложения при миграции в AWS.

16. **Backup and Restore** — процессы создания резервных копий и восстановления баз данных.
AWS RDS и Aurora предоставляют автоматическое резервное копирование и возможность
восстановления до определенного момента времени.

17. **Point-in-Time Recovery (PITR)** — функция, позволяющая восстанавливать базу данных
до состояния в любой момент времени в рамках заданного периода резервного копирования.

18. **Performance Insights** — инструмент для мониторинга производительности баз данных
RDS и Aurora. Позволяет анализировать рабочие нагрузки и выявлять узкие места.

19. **Enhanced Monitoring** — расширенный сбор метрик и мониторинг состояния базы данных
RDS и Aurora. Предоставляет детальную информацию о производительности инстансов.

20. **DB Parameter Group** — набор параметров конфигурации базы данных, который можно
настроить и применить к экземплярам RDS для управления поведением базы данных.

21. **DB Option Group** — набор дополнительных опций для базы данных RDS, таких как плагины
и модули, которые можно включить в конфигурацию базы данных.

22. **Read Scaling** — масштабирование базы данных для распределения нагрузки на чтение с
помощью Read Replica.

23. **Write Scaling** — масштабирование базы данных для распределения нагрузки на запись.
Может включать использование разделения базы данных и распределения нагрузки между
несколькими инстансами.

24. **Sharding** — метод горизонтального разделения базы данных на более мелкие, более
управляемые части, называемые шардами. Используется для улучшения производительности и
масштабируемости.

25. **Data Encryption** — шифрование данных на уровне базы данных для обеспечения безопасности.
Поддерживается на уровне томов EBS, шифрования данных в состоянии покоя и при передаче данных.
